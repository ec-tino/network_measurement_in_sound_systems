# network_measurement_in_sound_systems
This repo contains files used in a network measurement study: analyzing how stage sound networks handle heavy real-time data with low latency

## Main Project Questions:
- how many packets are transmitted within a specified time frame during a live show / no show?
- how do the sizes of the transmitted packets change depending on the network conditions?
    - Active network usage eg: during a live concert
    - Idle network eg: network without any live performance
    - Controlled network eg: intentionally controlling sound transmission for analysis.
 
## Phase 1: Capturing Packets Using tshark
- `capture.py` file contains code that builds a tshark command for capturing packets and writes the necessary fields (time, size, TP, IPs) of the captured packets into an output file that is later used for analysis.

- Run the command `sudo python capture.py --condition x --interface y --duration z` in your terminal to start capturing packets going through interface y under condition x for z seconds.
    - X: takes on 3 values: `live` | `idle` | `research`
    - Y: specifies the interface you want to capture packets on. eg en1 for WiFi, en0 for Ethernet, etc. NOTE: Use the command `tshark -D` to get a list of available interfaces on your device.
    - Z: how long you want to capture packets. 

## Phase 2: Analyzing Patterns in Captured Packets
- `analyze.py` file contains code that performs analysis on the captured packets. It takes as input the .csv file generated by `capture.py` and performs the following analysis:
    - compute the **summary statistics** of the packets. These include the total packets captured, the duration, packets per minute, total size of all captured packets, size of packets captured per minute, and the average packet size.
    - calculate the interpacket gap (IPG); this is the time between when two successive packets are captured. This data could be used as a proxy for latency in the network.
    - creates charts that visualize the summary statistics and ipg data.
 
- To conduct an analysis on your captured packets, run the command `python analyze.py directory_csv`, where `directory_csv` is the file path to your .csv file generated by `capture.py`. You can pass in multiple directories to this function; however, given that we are considering three conditions, pass in 3 directories at most. For example, `python analyze.py data/idle/file1.csv data/live/file1.csv data/research/file1.csv`.
